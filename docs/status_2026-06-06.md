A code-level audit of the deployed strategy, read against source rather than the docs.

## The correction that matters: "OOS 1.35" is selection-biased

The backtest handles the usual killers honestly. Fundamentals are captured at scan time and frozen
into the report row, not today's snapshot retro-applied; entry is real next-bar-open; cost and dedupe
are applied; and PF ≈ 1.35 reproduces deterministically from `trades.db`. But the headline number is
not a true holdout. `validate_quality_momentum.py` (`:22`, `:88-89`) re-slices the same full
`research_reports` history that `signal_search.py`'s 16-combo search already scored to crown
quality_momentum. The walk-forward is clean within that set, but its "OOS" windows were never held
out from the selection that picked the winner, so 1.35 carries multiple-comparisons selection bias.

Implication: 1.35 is an optimistic ceiling, not a validated edge. The live paper run since 6/04 is
the first genuine forward out-of-sample test. Watch ~30-50 main trades, not two weeks or a handful,
before trusting the number; real money stays off until live confirms. Open test so far: one closed
main trade (META −$558, stopped), which is noise.

## Fixed this session: high-VIX edge tightening (drift)

The regime block claimed, in its own comment and the project notes, that VIX>25 raises the edge
requirement to 3/3 "tighter in panic". In fact it only ever bumped min_conviction; `regime_min_edges`
stayed at the base value in every regime, so the panic edge-tightening was dead code and its log line
never fired. The logic is now extracted to a pure `DeepThinkAgent._regime_thresholds` staticmethod
that returns `min(3, base+1)` edges in high-vol, guarded by `tests/test_regime_thresholds.py` (7
cases). Behavior-preserving except the intended bump. It affects the penny book only, since main
ignores edges in signal_mode. Committed `bdb4dbd`, pushed to `RebelHawk-TK/DeepThinkTrader`.
